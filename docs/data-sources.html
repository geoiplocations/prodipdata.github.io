Documentation

Data sources and methodology inputs

All datasets used by ProdIPData are sourced from publicly available, authoritative internet and geographic data providers and are processed into a unified monthly release.

ProdIPData aggregates multiple authoritative public datasets and applies normalization, reconciliation, and classification layers to produce a consistent monthly IPv4 dataset.

Authoritative registries

Regional Internet Registries provide the foundation for allocation, delegation, and registry context.

  • RIPE NCC
  • ARIN
  • APNIC
  • LACNIC
  • AFRINIC

Routing intelligence

BGP data helps identify announced prefixes and ASN-to-prefix relationships.

  • RIPE RIS
  • RouteViews

Geolocation reference

GeoNames provides normalized geographic hierarchy, including countries, regions, cities, GeoName identifiers, and timezones.

Standardization

  • ISO country codes
  • IANA registries and RIR definitions
  • IANA Root Zone Database for internet namespace context

Special datasets

  • Bogon and reserved address-space references, including public bogon feeds such as Team Cymru where applicable.

ProdIPData processing layer

The final dataset is not a direct copy of any single source. It is produced through a controlled processing layer:

  • Cross-source reconciliation
  • ASN enrichment and classification
  • Geolocation normalization
  • RIR attribution based on ASN context rather than geolocation country
  • Consistency enforcement across monthly snapshots
